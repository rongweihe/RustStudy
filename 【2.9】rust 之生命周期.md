# 生命周期

- Rust的每个引用都有自己的生命周期
- 生命周期：引用保持有效的作用域
- 大多数情况下：生命周期是隐式的、可被推断的
- 当引用的生命周期可能以不同的方式互相关联的时候：手动标注生命周期。

生命周期可以说是 Rust 这门语言最大的特点

主要的作用：避免悬垂引用

## 生命周期的省略规则

在 Rust 引用分析中所编入的模式称为生命周期省略规则
- 这些规则无需开发者来遵守
- 它们是一些特殊情况，由编译器考虑
- 如果你的代码符合这些情况，那么就无需显示标注生命周期

生命周期省略规则不会提供完整的推断：
- 如果应用规则后，引用的生命周期仍然模糊不清-》编译错误
- 解决办法：添加生命周期标注，表明引用之间的相互关心
